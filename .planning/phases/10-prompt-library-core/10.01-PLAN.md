# Plan 10.01: Prompt Library Data Layer

<objective>
Build the prompt library data layer using a custom hook (`usePromptLibrary`) with localStorage persistence, curated presets, and CRUD operations.
</objective>

<must_haves>
- Custom hook `usePromptLibrary` handles all prompt state and localStorage integration.
- 3 curated presets are loaded by default on first access.
- UUID is used for generating IDs via `crypto.randomUUID()`.
- Max 50 prompts saved.
</must_haves>

<tasks>
<task>
<description>Define prompt types</description>
<read_first>
- src/types.ts
</read_first>
<action>
Add `SavedPrompt` interface to `src/types.ts`:
```typescript
export interface SavedPrompt {
  id: string;
  text: string;
  createdAt: number;
  isCurated?: boolean;
}
```
</action>
<acceptance_criteria>
- `src/types.ts` contains `export interface SavedPrompt`
- It has `id`, `text`, `createdAt`, and `isCurated` fields
</acceptance_criteria>
</task>

<task>
<description>Implement usePromptLibrary hook for data layer</description>
<read_first>
- src/hooks/usePromptLibrary.ts
</read_first>
<action>
Create `src/hooks/usePromptLibrary.ts`.
Initializes state from `localStorage` under key `chang_store_prompts`.
If empty, prepopulate with 3 curated presets ("Remove Hand from Pocket", "Untucked Shirt", "Combo") with `isCurated: true`.
Implement `savePrompt(text: string)` that assigns `crypto.randomUUID()`, sets `createdAt = Date.now()`, pre-pends to array, limits to 50 items.
Implement `deletePrompt(id: string)` that removes by id.
Implement `searchPrompts(query: string)` that returns filtered prompts ignoring case.
Store updates to `localStorage` on every change via `useEffect`.
</action>
<acceptance_criteria>
- `src/hooks/usePromptLibrary.ts` exports `usePromptLibrary`
- Uses `crypto.randomUUID()` for saving new prompts
- Prevents saving > 50 prompts
- Sorts newest first and handles curated prompts
- Saves to `chang_store_prompts` in localStorage
</acceptance_criteria>
</task>
</tasks>
