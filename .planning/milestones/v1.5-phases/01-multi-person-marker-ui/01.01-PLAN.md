# Plan 12.01: Multi-Person Hook State Extension

<objective>
Extend `useVirtualTryOn` with multi-person mode state: `isMultiPersonMode` boolean, `markerPosition` (normalized relative coords 0–1), and handlers `setIsMultiPersonMode` / `clearMarker`. No UI changes — pure state layer.
</objective>

<must_haves>
- `isMultiPersonMode: boolean` state defaults to `false`.
- `markerPosition: { x: number; y: number; relX: number; relY: number } | null` defaults to `null`. Pixel coords (`x`, `y`) used for rendering; relative coords (`relX`, `relY` in 0–1 range) reserved for Phase 13 engine use.
- `setIsMultiPersonMode(value: boolean)` — when called with `false`, also clears marker.
- `clearMarker()` — sets `markerPosition` to `null`.
- Uploading new subject images clears the marker (call `clearMarker` inside `handleSubjectImagesUpload`).
- All 4 new fields exported from the hook return object.
</must_haves>

```typescript
// Cuando se desactiva el modo, limpiar el marcador automáticamente

  if (!value) {
    setMarkerPosition(null);
  }

// Limpia el marcador sin cambiar el modo
const clearMarker = useCallback(() => {
  setMarkerPosition(null);
}, []);
```
