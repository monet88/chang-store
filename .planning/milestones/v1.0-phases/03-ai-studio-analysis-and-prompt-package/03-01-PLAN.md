<task type="auto">
  <name>Task 2: Create the upscale analysis service with Gemini structured output and prompt builder</name>
  <files>services/upscaleAnalysisService.ts</files>
  <action>
Create `services/upscaleAnalysisService.ts` with two exports:

**1. `analyzeImageForUpscale(image: ImageFile, model?: string): Promise<UpscaleAnalysisReport>`**

Implementation:
- Import `Part`, `Type` from `@google/genai`, `getGeminiClient` from `apiClient`, `logApiCall` from `debugService`, and the report types from `types`.
- Default model to `'gemini-2.5-flash'`.
- Build an `imagePart` (inlineData with base64 + mimeType) and a `textPart` with a system prompt:

```
# ROLE
You are a Fashion Photography Analysis AI for upscale preparation.

# OBJECTIVE
Analyze the provided fashion photograph and produce a structured JSON report covering garments, materials, background, lighting, framing, pose, and preservation risks. This report will be used to generate an upscale prompt that faithfully preserves all details.

# ANALYSIS DIMENSIONS
1. **Garments**: Identify each visible garment — name, type (dress/top/pants/etc), description of cut, construction, and styling.
2. **Materials**: For each garment, describe the fabric type, perceived texture, weight (light/medium/heavy), and sheen (matte/semi-gloss/high-sheen).
3. **Background**: Describe the environment, visible surfaces, depth perception, and overall background.
4. **Lighting**: Describe key light direction, quality (soft/hard), color temperature (warm/neutral/cool), and shadow behavior.
5. **Framing**: Identify shot type (full-body/half-body/close-up), camera angle, and composition approach.
6. **Pose**: Describe body position, hand/arm gesture, facial expression, and movement quality (static/implied motion).
7. **Preservation Risks**: Identify areas with fine detail that are at risk of loss or distortion during upscaling — text/logos, fine jewelry, embroidery, intricate patterns, hair strands, skin texture. Rate each as high/medium/low risk.

# OUTPUT
Return ONLY the structured JSON matching the required schema. No markdown, no explanations.
```

- Call `ai.models.generateContent()` with:
  - `model`: the resolved model
  - `contents: { parts: [imagePart, textPart] }`
  - `config.responseMimeType: "application/json"`
  - `config.responseSchema`: a full `Type.OBJECT` schema matching `UpscaleAnalysisReport` — define every field with `Type.STRING`, arrays with `Type.ARRAY`, nested objects with `Type.OBJECT`. Mark garments, materials, preservationRisks as arrays. Mark background, lighting, framing, pose as objects. Set `riskLevel` as `Type.STRING` with description "high, medium, or low".
  - `config.thinkingConfig: { thinkingBudget: 16384 }`
- Follow existing safety block/candidate validation from `gemini/text.ts`.
- Parse response text as JSON, validate it has the expected shape (garments array, etc.).
- Log via `logApiCall({ provider: 'Gemini', model, feature: 'Upscale Analysis', ... })`.
- Return the parsed `UpscaleAnalysisReport`.

**2. `generateUpscalePrompt(report: UpscaleAnalysisReport, quality: UpscaleQuality): string`**

This is a **pure function** (no API call):
- Build a prompt paragraph from the report data:
  - Start: `"Upscale this fashion photograph to {2K/4K} resolution."`
  - Garments: `"Preserve the {garment1.name}: {garment1.description}; {garment2.name}: {garment2.description}."`
  - Materials: `"Maintain fabric textures — {material1.fabric} ({material1.texture}, {material1.sheen}), ..."`
  - Lighting: `"Keep the {lighting.quality} lighting from {lighting.direction} with {lighting.colorTemperature} color temperature."`
  - Pose: `"Retain the subject's {pose.bodyPosition} with {pose.expression} expression."`
  - Preservation: For each high or medium risk item: `"CRITICAL PRESERVATION: {risk.area} — {risk.detail}."`
  - End: `"Photorealistic, fashion photography quality, {quality} quality."`
- Return the composed string.
  </action>
  <verify>npm run build</verify>
  <done>Analysis service calls Gemini with structured JSON schema for fashion analysis. Prompt builder composes preservation-first prompt from report data without a second API call.</done>
