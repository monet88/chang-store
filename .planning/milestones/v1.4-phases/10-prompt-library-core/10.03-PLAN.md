# Plan 10.03: UAT Gap Closure - Title, Expand, Copy, and Search Behavior

## Objective
Update the Prompt Library to support prompt titles, expand/collapse functionality for content viewing, a copy button, and restrict search filtering strictly to the title.

## Context
During UAT, the user identified two primary issues:
1. Prompts need a dedicated title, and clicking a prompt should expand its full content along with a "Copy prompt" button, rather than just returning/inserting it immediately.
2. The search functionality should only filter based on the title, not the actual prompt content.

## Tasks

1. **Update Data Layer**
   - Modify `SavedPrompt` interface in `src/types.ts` to include an optional (or required) `title?: string` field.
   - Update `usePromptLibrary.ts` to ensure default curated prompts have a distinct `title`.
   - Modify `searchPrompts` in `usePromptLibrary.ts` to search exclusively against `prompt.title` (falling back to `prompt.text` only if `title` is missing for legacy/migration safety).

2. **Update UI Component (`PromptLibraryModal.tsx`)**
   - Modify the rendering of each list item to show `prompt.title` prominently if available.
   - Introduce an expanded state (e.g. `expandedPromptId`) to track which prompt is currently opened.
   - When expanded, display the full `prompt.text` inside a dedicated block and show a "Copy" button.
   - Using the `navigator.clipboard.writeText` API, copy the prompt.text to clipboard and optionally show a toast/success indicator.
   - Keep the "Delete" button visible when applicable.
   - The `onSelect` behavior (inserting the prompt) can be bound to an "Sử dụng" (Use) button instead of replacing the entire row click action.

## Validation
- Saving/fetching prompts with titles runs without errors.
- Text search yields results only if the search string exists within the title.
- Clicking a prompt expands it, showing full content and a working Copy button.
